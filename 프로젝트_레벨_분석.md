## 1. game-server (C++ 게임 서버 기초부터 프로덕션까지)

### 📋 프로젝트 개요
- **Track A**: gameserver-fundamentals (Lab 1.1-1.4) - TCP/WebSocket 기반 기초
- **Track B**: netcode-core (v1.0+) - UDP 권위 서버 + 스냅샷/델타

### 👨‍💻 필요한 개발자 수준

**Track A (Lab 1.1-1.4)**: **중급 개발자** (경력 1-3년)
- C++ 메모리 관리 (RAII, 스마트 포인터) 이해 필요
- 네트워크 프로그래밍 기초 (소켓, TCP/UDP)
- 멀티스레딩 개념
- **학습 가능**: CS 전공자라면 충분히 학습하며 구현 가능

**Track B (netcode-core v1.0+)**: **고급 개발자** (경력 3-5년+)
- 저수준 네트워킹 깊이 (UDP 신뢰성 레이어 직접 구현)
- 게임 물리/동기화 이론 (클라이언트 예측, 리컨실리에이션)
- 성능 최적화 (60 TPS 유지, p99 latency < 15ms)
- Prometheus/Grafana 모니터링 설계
- **난이도**: 상당한 배경 지식 필요

### ⏱️ 구현 기간

**Track A**:
- Lab 1.1 (TCP Echo): 1주
- Lab 1.2 (Turn Combat): 1주
- Lab 1.3 (WebSocket Chat): 1주
- Lab 1.4 (Real-time Pong): 1주
- **총 4주**

**Track B**:
- v1.0 (UDP Core): 2-3주
- v1.1 (Snapshot/Delta): 2-3주
- v1.2 (Observability): 2주
- v1.3 (Match/Room): 2-3주
- **총 8-11주**

**전체 프로젝트**: **12-16주** (3-4개월)

### 🎯 포트폴리오 경쟁력

**신입 지원 시**: ⭐⭐⭐⭐⭐ (최상급)
- 대부분의 신입이 유니티 MLAPI/Netcode를 사용
- UDP 신뢰성 레이어를 직접 구현한 경우는 극히 드물음
- C++ 깊이와 네트워킹 이해도를 명확히 입증
- **차별화**: 상위 1% 수준

**이직 지원 시**: ⭐⭐⭐⭐⭐ (최상급)
- Nexon/Krafton/Netmarble 등 게임 서버 포지션에 최적
- 프로덕션 품질 (CI/CD, 모니터링, 부하 테스트)
- 실무에서 바로 활용 가능한 수준
- **경쟁력**: 면접 확정 수준

### 💡 추천 대상
- 게임 서버 개발자 희망자 (필수)
- C++ 저수준 프로그래밍 깊이를 보여주고 싶은 개발자
- 네트워크 프로그래밍 전문성 입증 필요자

## 3. video-editor (VrewCraft - 웹 기반 동영상 편집기)

### 📋 프로젝트 개요
- React 18 + Node.js 20 + C++ N-API
- FFmpeg C API 직접 사용
- 썸네일 추출, 메타데이터 분석 (p99 < 50ms)
- Prometheus/Grafana 모니터링

### 👨‍💻 필요한 개발자 수준

**중급~고급 개발자** (경력 2-5년)
- **프론트엔드**: React 18, TypeScript 5, 현대적 웹 개발
- **백엔드**: Node.js 20, WebSocket, 비동기 처리
- **C++**: N-API, FFmpeg C API, RAII 메모리 관리
- **멀티 스킬**: 웹 + 저수준 C++ 동시 구현
- **난이도**: 매우 높음 (웹과 C++ 깊이 모두 필요)

### ⏱️ 구현 기간

**Phase 1** (편집 기능): 4-5주
- React UI, 비디오 업로드, 타임라인, 트림/분할/자막

**Phase 2** (C++ 성능): 4-5주
- N-API 네이티브 애드온
- FFmpeg C API 통합
- 썸네일/메타데이터 추출 최적화

**Phase 3** (프로덕션 완성도): 3-4주
- Docker 배포
- Prometheus/Grafana
- 포괄적 문서화

**총 기간**: **12-14주** (3-3.5개월)
**현재 상태**: Phase 3 완료 ✅

### 🎯 포트폴리오 경쟁력

**신입 지원 시**: ⭐⭐⭐⭐⭐
- **대부분의 신입**: FFmpeg 래퍼 사용
- **이 프로젝트**: C API 직접 구현, RAII 메모리 관리
- **차별화**: 웹 개발 + 저수준 C++ 전문성 동시 입증

**이직 지원 시**: ⭐⭐⭐⭐⭐
- **100% Voyager X (Vrew) 기술 스택 매칭**
- React, Node.js, TypeScript, C++, FFmpeg 모두 활용
- "필요에 따라서 더욱 저수준으로 내려갈 수 있음" 완벽 입증
- **경쟁력**: Vrew 지원 시 최상위 후보

### 💡 추천 대상
- Voyager X (Vrew) 지원자 (필수)
- 웹 + C++ 멀티 스킬 입증 필요자
- 비디오 처리 도메인 관심자

- 면접 주제 독점: "웹 프레임워크를 처음부터 구현했습니다"
- 의존성 주입, 연결 풀링 내부 동작 설명 가능
- Spring Boot/Django 등 어떤 프레임워크도 깊이 이해
- **경쟁력**: 모든 백엔드 포지션에 강력한 어필

### 💡 추천 대상

- **독립 프로젝트**: 병렬 불가 (각각 집중 필요)
- **연관 프로젝트**: 순차 진행 권장
  - 예: framework → backend/spring
  - 예: game-server Track A → Track B
