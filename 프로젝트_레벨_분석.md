# 프로젝트 레벨 및 경쟁력 분석

각 프로젝트의 구현 난이도, 필요한 개발자 수준, 예상 기간, 그리고 포트폴리오로서의 경쟁력을 분석합니다.

## 📖 프로젝트 이해하기

각 프로젝트에 대한 **가장 정확하고 상세한 정보**는 해당 프로젝트의 `design/` 폴더에 있습니다:

- **arena/design/**: MVP 1.0-1.3 설계 문서 (게임 서버, 전투, 매치메이킹, 통계)
- **backend/node.js/design/**: N2.0-N2.6 설계 문서 (NestJS 패턴 훈련)
- **backend/spring/design/**: Milestone 2.0-2.6 설계 문서 (Spring Boot 패턴 훈련)
- **e-commerce/design/**: Stage 0-4 설계 문서 (전자상거래 플랫폼)
- **framework/design/**: Milestone 1.1-1.4.5 설계 문서 (프레임워크 기초)
- **game-server/design/**: Lab 1.1-1.4 및 netcode-core 1.0-1.3 설계 문서
- **react/design/**: Milestone 1-6 설계 문서 (React 패턴 훈련)
- **video-editor/design/**: Phase 0.0-3.0 설계 문서 (동영상 편집기)

**중요**: 이 문서의 정보와 각 프로젝트의 design/ 문서가 상충할 경우, **design/ 문서가 정합**합니다.

## 1. game-server (C++ 게임 서버 기초부터 프로덕션까지)

### 📋 프로젝트 개요
- **Track A**: gameserver-fundamentals (Lab 1.1-1.4) - TCP/WebSocket 기반 기초
- **Track B**: netcode-core (v1.0-1.3) - UDP + Protobuf + 매칭 시스템

### 👨‍💻 필요한 개발자 수준

**Track A (Lab 1.1-1.4)**: **주니어~미들 개발자** (경력 0-2년)
- C++ 메모리 관리 기초 (RAII, 스마트 포인터)
- 네트워크 프로그래밍 기초 (소켓, TCP/WebSocket)
- 멀티스레딩 개념
- **학습 가능**: CS 전공자라면 충분히 학습하며 구현 가능
- **난이도**: 낮음~중간 (교육용 프로젝트)

**Track B (netcode-core v1.0-1.3)**: **미들~시니어 개발자** (경력 2-5년)
- Boost.Asio UDP 네트워킹
- Protobuf 직렬화
- 게임 상태 동기화 패턴
- Prometheus 모니터링
- **난이도**: 중상 (실전 네트워크 엔진, 하지만 arena보다 덜 완성됨)

### ⏱️ 구현 기간 (design/ 기반 실측)

**Track A** (교육용 랩):
- Lab 1.1-1.4: 각 3-4일
- **총 2주**

**Track B** (실전 엔진):
- v1.0-v1.3: 각 1-1.5주
- **총 4-6주**

**전체 프로젝트**: **6-8주** (1.5-2개월)

### 🎯 포트폴리오 경쟁력

**신입 지원 시**: ⭐⭐⭐⭐ (상급)
- **차별화 근거**: C++ 네트워크 엔진 구현 경험
- Track A: 교육적 가치는 있으나 차별화 부족
- Track B: Boost.Asio + Protobuf 실전 경험 입증
- **한계**: arena보다 완성도 낮음 (완전한 게임 없음)

**이직 지원 시**: ⭐⭐⭐⭐ (상급)
- **타겟**: Nexon/Krafton/Netmarble 게임 서버 포지션
- **검증된 기술**: C++ 네트워킹, 상태 동기화, 매칭 시스템
- Prometheus 모니터링, 부하 테스트 경험
- **경쟁력**: 게임 서버 포지션에 강력하나, arena가 더 완성도 높음

### 💡 추천 대상
- 게임 서버 개발자 희망자 (필수)
- C++ 저수준 프로그래밍 깊이를 보여주고 싶은 개발자
- 네트워크 프로그래밍 전문성 입증 필요자

## 2. arena (실시간 1v1 대결 게임 서버)

### 📋 프로젝트 개요
- 60 TPS WebSocket 게임 서버
- ELO 매치메이킹, 전투 시스템, 리더보드
- PostgreSQL, Redis, Prometheus 통합

### 👨‍💻 필요한 개발자 수준

**시니어 개발자** (경력 3-5년)
- C++17 모던 C++ 숙련 (RAII, 스마트 포인터, 이동 시맨틱스)
- Boost.Asio/Beast 비동기 I/O 실전 경험
- 60 TPS 고정 타임스텝 게임 루프 구현 능력 (분산 < 1ms)
- 실시간 멀티플레이어 네트워킹 이해
- 데이터베이스 설계 및 성능 최적화
- **난이도**: 높음 (프로덕션 품질 멀티플레이어 서버)

### ⏱️ 구현 기간 (design/ 기반 실측)

**MVP 1.0** (기본 게임 서버): 1.5주
**MVP 1.1** (전투 시스템): 1.5주
**MVP 1.2** (매치메이킹): 2주
**MVP 1.3** (통계 & 랭킹): 2주

**총 기간**: **6-8주** (1.5-2개월)
**현재 상태**: Checkpoint A 완료 ✅

### 🎯 포트폴리오 경쟁력

**신입 지원 시**: ⭐⭐⭐⭐⭐
- **차별화 근거**: 대부분의 신입은 게임 서버를 Unity/Unreal 엔진으로 구현
- 이 프로젝트: **from scratch C++ 멀티플레이어 서버** (WebSocket, 60 TPS, ELO)
- 성능 벤치마크 문서화 (틱 분산 < 1ms, p99 지연 < 20ms)
- 완성도: 실제 플레이 가능한 1v1 대전 게임

**이직 지원 시**: ⭐⭐⭐⭐⭐
- **타겟**: Nexon, Krafton, Netmarble, Kakao Games
- **검증된 기술**: 실시간 네트워킹, 상태 동기화, ELO 매칭
- 프로덕션 품질 (Prometheus, 21개 테스트, CI/CD)
- **경쟁력**: 게임 서버 포지션에 매우 강력

### 💡 추천 대상
- 게임 서버 개발자 (특히 한국 게임사)
- 완성도 높은 포트폴리오 필요자
- 실제 구동되는 프로젝트 선호자

## 3. video-editor (VrewCraft - 웹 기반 동영상 편집기)

### 📋 프로젝트 개요
- React 18 + Node.js 20 + C++ N-API
- FFmpeg C API 직접 사용
- 썸네일 추출, 메타데이터 분석 (p99 < 50ms)
- Prometheus/Grafana 모니터링

### 👨‍💻 필요한 개발자 수준

**중급~고급 개발자** (경력 2-5년)
- **프론트엔드**: React 18, TypeScript 5, 현대적 웹 개발
- **백엔드**: Node.js 20, WebSocket, 비동기 처리
- **C++**: N-API, FFmpeg C API, RAII 메모리 관리
- **멀티 스킬**: 웹 + 저수준 C++ 동시 구현
- **난이도**: 매우 높음 (웹과 C++ 깊이 모두 필요)

### ⏱️ 구현 기간 (design/ 기반 실측)

**Bootstrap** (Phase 0.0): 1주

**Phase 1** (편집 기능, 1.0-1.3): 3주
- React UI, 비디오 업로드, 타임라인, 트림/분할/자막

**Phase 2** (C++ 성능, 2.0-2.3): 4-5주
- N-API 네이티브 애드온
- FFmpeg C API 통합
- 썸네일/메타데이터 추출 최적화

**Phase 3** (프로덕션, 3.0): 2-3주
- Docker 배포, Prometheus/Grafana, 포괄적 문서화

**총 기간**: **10-12주** (2.5-3개월)
**현재 상태**: Phase 3 완료 ✅

### 🎯 포트폴리오 경쟁력

**신입 지원 시**: ⭐⭐⭐⭐⭐
- **차별화 근거**: 대부분의 웹 개발자는 FFmpeg 래퍼 라이브러리 사용
- 이 프로젝트: **FFmpeg C API 직접 사용 + N-API 네이티브 애드온**
- RAII 메모리 관리로 메모리 릭 0 달성
- **멀티 스킬**: React + Node.js + C++ (매우 드묾)

**이직 지원 시**: ⭐⭐⭐⭐⭐
- **타겟**: Voyager X (Vrew) - 웹 애플리케이션 개발자
- **기술 스택 100% 매칭**: React, Node.js, TypeScript, C++, FFmpeg
- "필요에 따라서 더욱 저수준으로 내려갈 수 있음" 완벽 입증
- **경쟁력**: Vrew 지원 시 최상위 후보 (기술 스택 완벽 매칭)

### 💡 추천 대상
- Voyager X (Vrew) 지원자 (필수)
- 웹 + C++ 멀티 스킬 입증 필요자
- 비디오 처리 도메인 관심자

## 4. e-commerce (Sagaline - 전자상거래 플랫폼)

### 📋 프로젝트 개요
- Spring Boot 3.2, Java 17
- PostgreSQL, Redis, Kafka, Elasticsearch
- 모놀리스 → 마이크로서비스 진화
- 한국 시장 (Toss Payments, Kakao OAuth, PIPA)

### 👨‍💻 필요한 개발자 수준

**Stage 1-4 (Checkpoint: Core)**: **주니어~중급** (경력 0-2년)
- Spring Boot 기본 이해
- REST API, CRUD, 트랜잭션
- JPA/Hibernate 기초
- **학습 가능**: 전공자 또는 부트캠프 수료자 가능

**Stage 5 (Checkpoint: Scale)**: **중급** (경력 2-3년)
- 캐싱 전략 (Redis)
- 비동기 처리 (Kafka)
- 검색 엔진 (Elasticsearch)

**Stage 6-9 (Checkpoint: Cloud)**: **중급~고급** (경력 3-5년)
- 마이크로서비스 아키텍처
- Kubernetes, 서비스 메시 등
- 분산 시스템 설계

### ⏱️ 구현 기간

**Checkpoint: Core** (Stage 1-4)
- Stage 1 (모놀리스): 4주
- Stage 2 (관측성): 3주
- Stage 3 (규모 확장): 6주
- Stage 4 (안정성): 4주
- **소계**: **18주** (4.5개월)

**Checkpoint: Scale** (+ Stage 5): **+7주** (총 25주)

**Checkpoint: Cloud** (+ Stage 6-9): **+11주** (총 36주 = 9개월)

**현재 상태**: Bootstrap 완료

### 🎯 포트폴리오 경쟁력

**신입 지원 시 (Core 완료 기준)**: ⭐⭐⭐⭐
- 전형적인 백엔드 기술 스택
- 실무 수준의 전자상거래 도메인
- CI/CD, 테스트 커버리지 ≥80%
- **차별화**: 한국 시장 통합 (Toss, Kakao)

**이직 지원 시 (Scale 완료 기준)**: ⭐⭐⭐⭐⭐
- Kafka, Elasticsearch, Redis 실전 경험
- 캐싱, 검색, 비동기 처리 패턴
- **경쟁력**: 미들 레벨 백엔드 포지션

**이직 지원 시 (Cloud 완료 기준)**: ⭐⭐⭐⭐⭐
- 마이크로서비스 아키텍처
- Kubernetes, 클라우드 네이티브
- **경쟁력**: 시니어 백엔드 포지션

### 💡 추천 대상
- 백엔드 개발자 지망생 (신입~경력)
- Spring Boot 깊이 있는 학습 희망자
- 전자상거래 도메인 경험 필요자
- 주니어 → 미들 → 시니어 경로 구축

- Node.js 백엔드 개발자 지망생
- NestJS 패턴 학습 희망자

- 스타트업 (Tier 3)에서는 단독으로도 강력
- Tier 1-2에서는 **framework 프로젝트와 결합 필수**

**지원 전략**:
1. **단독 지원**: Tier 3 스타트업 (기본기 증명)
2. **framework + backend 조합**: Tier 1-2 (이론 + 실전 스토리)
   - "framework에서 배운 내부 동작을, Spring Boot/NestJS로 실무 적용"
3. **면접 준비**:
   - "왜 NestJS와 Spring 둘 다 했나요?" → "패턴의 공통점과 차이점 비교 학습"

---

### 7. react (React 웹 패턴 훈련)

**타겟 회사 (Tier 2-3)**:
- **프론트엔드 주니어 포지션**
- backend 프로젝트와 결합하여 **풀스택 개발자** 지원

**지원 전략**:
1. **단독 지원**: 차별화 부족, Tier 3에서만 가능
2. **풀스택 조합**:
   - react + backend/node.js → Node.js 풀스택 개발자
   - react + backend/spring → Spring 풀스택 개발자
3. **면접 준비**: "프론트엔드와 백엔드 모두 할 수 있습니다" 실질적 증명

## 📝 참고 사항

### 프로젝트 품질 기준
모든 프로젝트는 다음을 포함:
- ✅ CI/CD 파이프라인
- ✅ 테스트 커버리지 (≥70-80%)
- ✅ 포괄적 문서화
- ✅ 성능 벤치마크 검증
- ✅ 프로덕션 수준 인프라 (Docker, PostgreSQL, Redis 등)

### 학습 vs 구현 시간
- **순수 구현 시간**: 위 표의 기간
- **학습 포함 시간**: 1.5-2배 예상
- **처음 접하는 기술**: 추가 2-4주 학습 기간

### 병렬 작업 가능성
- **독립 프로젝트**: 병렬 불가 (각각 집중 필요)
- **연관 프로젝트**: 순차 진행 권장
  - 예: framework → backend/spring
  - 예: game-server Track A → Track B
