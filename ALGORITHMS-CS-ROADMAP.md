# ROADMAP for Algorithms & CS

> **목적**: 코딩테스트(백준 골드 상위) + 기술 면접 대응력 확보
> **특징**: 프로젝트와 독립적인 별도 트랙, 효율성 중심
> **범위**: 골드 1~3 수준 알고리즘 + 실무 면접 필수 CS 기초

---

## 📚 전체 구조

이 로드맵은 두 개의 독립적인 트랙으로 구성됩니다:

* **트랙 A – 알고리즘/코딩테스트** (백준 골드 상위 목표)
* **트랙 B – CS 기초** (OS/네트워크/DB 면접 대응)

### 추천 학습 순서

```
A0 (준비) → A1 (실버) → A2 (골드) + B1/B2 병행 → A3 (선택 심화) + B3
```

**핵심 원칙**:
1. 모든 것을 마스터하려 하지 말 것
2. 필요한 것만, 실제 쓸 수 있는 수준까지만
3. 프로젝트 경험과 연결하여 설명력 확보

---

## 🎯 목표 기준

### 코딩테스트
- **최소 목표**: 백준 골드 4~5 안정적 통과
- **목표**: 골드 1~3 접근 가능
- **상한선**: 플래티넘 맛보기 (선택)

### 기술 면접
- **OS**: 프로세스/스레드/락/메모리 구조를 본인 말로 설명
- **네트워크**: HTTP/TCP/UDP/WebSocket을 프로젝트 기준으로 설명
- **DB**: 인덱스/트랜잭션/락을 본인 프로젝트 DB 설계로 설명

```

```
평일: 알고리즘 문제 풀이 (A1→A2)
주말: CS 이론 학습 (B1→B2→B3)
```

### 패턴 3: 프로젝트 중심
```
프로젝트 개발 중: CS 이론 병행 (B1-B3)
프로젝트 완료 후: 알고리즘 집중 (A1-A3)

## 🎓 프로젝트 연계 전략

### 알고리즘 → 프로젝트
- **그래프 알고리즘** → video-editor 작업 의존성 그래프
- **동적 계획법** → 캐시 최적화 패턴
- **세그먼트 트리** → 게임 서버 구간 쿼리

### CS 이론 → 프로젝트
- **OS (스레드/락)** → C++ 게임 서버 동시성 제어
- **네트워크 (TCP/UDP)** → 넷코드 프로토콜 설계
- **DB (인덱스/트랜잭션)** → NestJS 백엔드 쿼리 최적화

### 면접 대응
각 CS 모듈을 학습할 때 다음 질문에 답할 수 있도록:

**"본인 프로젝트에서 이 기술을 어떻게 사용했나요?"**

예:
- "게임 서버에서 UDP를 선택한 이유는..."
- "Redis 캐시를 도입해서 조회 성능을 X배 개선했습니다..."
- "PostgreSQL 인덱스를 추가해서 쿼리 시간을 Y초에서 Z초로 단축했습니다..."

## 💡 효율적 학습 팁

### 알고리즘 (트랙 A)
1. **템플릿 코드 준비**: BFS/DFS/세그먼트 트리 등 자주 쓰는 코드 미리 작성
2. **패턴 인식 훈련**: 문제 유형을 빠르게 파악하는 연습
3. **시간 제한**: 실버 30분, 골드 1시간 안에 풀기 연습
4. **복습 루틴**: 틀린 문제는 3일 후, 1주 후 다시 풀기

### CS 기초 (트랙 B)
1. **그림 그리기**: 개념을 종이에 그려가며 설명 연습
2. **프로젝트 연결**: 배운 내용을 본인 프로젝트 코드에서 찾기
3. **모의 면접**: 개념을 5분 안에 설명하는 연습
4. **정리 노트**: A4 1장 분량으로 핵심만 요약
