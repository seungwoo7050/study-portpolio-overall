# 문서 1. 전체 아키텍처 & 로드맵 개요

## 1. 이 문서의 역할

이 문서는 **"전체 그림"만 보여주는 인덱스 문서**다.

* 대상: 기본적인 프로그래밍은 할 수 있지만, React/TypeScript로 실전 웹 애플리케이션을 만들어본 적 없는 개발자
* 목표:

  * React + TypeScript 웹 애플리케이션의 **핵심 패턴과 구조**를 한 번에 이해
  * 어떤 순서(Milestone)로 문서 2, 3, 4를 보면 되는지 파악
  * 각 Milestone에서 **무엇을 만들고, 통과 기준이 뭔지** 명확히 정리

세부 문법, 코드, 설정은 각각:

* **문서 2**: React 기초 (M1-M2: Bootstrap + CRUD)
* **문서 3**: 인증 & 상태관리 (M3-M4: Auth + Data Fetching)
* **문서 4**: 고급 패턴 (M5-M6: Search/Performance + Testing)

에서 다룬다. 이 문서는 그 위에 얹는 "지도"라고 생각하면 된다.

---

## 2. 최종 목표: 어떤 애플리케이션을 만들 것인가

최종적으로 목표하는 것은:

> **실전에서 자주 쓰이는 React + TypeScript 프론트엔드 패턴을 단일 프로젝트에서 점진적으로 익히는 것**

구체적으로:

1. **프로젝트 기반 구축** (M1)

   * Vite + React 19 + TypeScript
   * React Router v6 라우팅
   * Tailwind CSS 스타일링
   * GitHub Actions CI 파이프라인

2. **CRUD UI 구현** (M2)

   * Issue Tracker 도메인
   * React Hook Form + Zod 검증
   * API 연동 (fetch/axios)
   * 목록/상세/생성/수정/삭제 화면

3. **인증/인가 UX** (M3)

   * JWT 기반 로그인
   * Protected Routes
   * 역할 기반 UI (OWNER/MANAGER/MEMBER)
   * Context API 또는 상태관리 라이브러리

4. **데이터 패칭 패턴** (M4)

   * React Query (TanStack Query)
   * 캐싱 & Optimistic Update
   * 통계 대시보드
   * 차트 라이브러리 (Recharts)

5. **검색/필터 & 성능** (M5)

   * 검색 UI (디바운스)
   * 필터링 (카테고리, 가격 범위)
   * 가상 스크롤 (react-window)
   * Code Splitting & Lazy Loading

6. **테스트 & 접근성** (M6)

   * Vitest + React Testing Library
   * E2E (Cypress/Playwright)
   * ARIA 레이블
   * 키보드 네비게이션

**이 전체 흐름을 Milestone 1 → 6으로 쪼개서 올라간다.**

## 3. 전체 아키텍처 한 번에 보기

### 3.1 기술 스택

```text
[React 19 + TypeScript]
  - Vite (빌드 도구)
  - React Router v6 (라우팅)
  - Tailwind CSS (스타일링)
        │
        ├─ React Hook Form + Zod (폼 검증)
        ├─ React Query (데이터 패칭 & 캐싱)
        ├─ Recharts (차트)
        ├─ react-window (가상 스크롤)
        └─ Vitest + React Testing Library (테스트)
```

### 3.2 프로젝트 구조

```text
frontend/
├── src/
│   ├── main.tsx                   # 엔트리포인트
│   ├── app/
│   │   ├── App.tsx                # 라우터 루트
│   │   ├── routes/                # 라우트 정의
│   │   └── layouts/               # 레이아웃 컴포넌트
│   ├── shared/
│   │   ├── components/            # 공통 UI 컴포넌트
│   │   ├── hooks/                 # 커스텀 훅
│   │   └── lib/                   # 유틸리티
│   │       ├── apiClient.ts       # API 클라이언트
│   │       └── queryClient.ts     # React Query 설정
│   └── features/
│       ├── auth/                  # 인증 (M3)
│       ├── user/                  # 사용자 관리
│       ├── project/               # 프로젝트 (M2)
│       ├── issue/                 # 이슈 (M2)
│       ├── comment/               # 댓글 (M2)
│       ├── team/                  # 팀 (M3)
│       ├── stats/                 # 통계 (M4)
│       └── search/                # 검색 (M5)
├── tests/
│   ├── unit/                      # 단위 테스트 (M6)
│   └── e2e/                       # E2E 테스트 (M6)
└── .github/
    └── workflows/
        └── ci.yml                 # CI 파이프라인 (M1)
```

### 3.3 대표 시나리오 (Issue Tracker)

```text
1. 사용자 로그인: POST /api/auth/login → JWT 토큰 저장
2. 프로젝트 목록 조회: GET /api/projects (React Query 캐싱)
3. 이슈 생성: POST /api/projects/{id}/issues (React Hook Form 검증)
4. 이슈 목록 조회: GET /api/projects/{id}/issues (무한 스크롤)
5. 이슈 상세: GET /api/issues/{id} (댓글 포함)
6. 댓글 작성: POST /api/issues/{id}/comments (Optimistic Update)
7. 통계 대시보드: GET /api/stats/daily (차트 렌더링)
8. 검색: GET /api/search/products?q=키워드 (디바운스 입력)
```

## 4. Milestone별 로드맵 개요

Milestone은 **"검증 가능한 기능 단위"** 기준으로 나눈다.

* **Milestone 1**: Bootstrap & Routing & CI
* **Milestone 2**: Issue Tracker CRUD
* **Milestone 3**: 로그인/인증/인가 UX
* **Milestone 4**: 통계/데이터 패칭 패턴
* **Milestone 5**: 검색/필터 UI + 성능 기초
* **Milestone 6**: 테스트 전략 & E2E & 접근성

각 Milestone마다:

* 필요 기술
* 만들어야 할 최소 기능
* 통과 체크리스트

를 명확하게 정의한다.

## 5. Milestone 1-2 – React 기초 (Bootstrap + CRUD)

### 5.1 목표

* Vite + React + TypeScript 프로젝트를 생성하고,
* React Router로 **라우팅** 구조를 만들 수 있으며,
* **CRUD UI**를 React Hook Form + Zod로 구현할 수 있는 수준.

이 단계에서는 **인증, React Query, 성능 최적화는 아직 안 건드린다.**
목표는 "React의 기본 패턴에 적응하는 것".

### 5.2 다루는 기술

* **M1 - Bootstrap & Routing & CI**

  * Vite 프로젝트 생성
  * React Router v6 (Routes, Route, Link, useNavigate)
  * Tailwind CSS 설정
  * 레이아웃 컴포넌트 (Header, Sidebar)
  * GitHub Actions CI

* **M2 - Issue Tracker CRUD**

  * React Hook Form (useForm, Controller)
  * Zod 스키마 검증
  * API 연동 (fetch/axios)
  * useState, useEffect 기본 패턴
  * 목록/상세/생성/수정/삭제 화면

### 5.3 이 단계에서 완성해야 할 기능

**M1 최소 기능**:

1. **프로젝트 생성 & 라우팅**

   * `npm create vite@latest` - React + TypeScript
   * 기본 라우트 (`/`, `/login`, `/projects`, `/issues`)
   * 레이아웃 컴포넌트 (Header, Sidebar)

2. **공통 컴포넌트**

   * Button, Input, Card 등 재사용 가능한 UI

3. **CI 파이프라인**

   * GitHub Actions: `npm test`, `npm run build` 자동 실행

**M2 최소 기능**:

1. **이슈 CRUD**

   * `GET /api/projects` - 프로젝트 목록
   * `POST /api/projects/{id}/issues` - 이슈 생성 (React Hook Form)
   * `GET /api/issues/{id}` - 이슈 상세
   * `PUT /api/issues/{id}` - 이슈 수정
   * `DELETE /api/issues/{id}` - 이슈 삭제

2. **폼 검증**

   * Zod 스키마로 클라이언트 검증
   * 에러 메시지 표시

### 5.4 통과 체크리스트

* [ ] Vite 프로젝트를 생성하고, `npm run dev`로 로컬에서 실행할 수 있다.
* [ ] React Router로 여러 페이지를 만들고, 네비게이션할 수 있다.
* [ ] Tailwind CSS로 레이아웃과 스타일을 적용할 수 있다.
* [ ] React Hook Form + Zod로 폼을 만들고, 검증할 수 있다.
* [ ] fetch/axios로 백엔드 API를 호출하고, 응답을 화면에 표시할 수 있다.
* [ ] GitHub Actions CI가 push 시마다 자동으로 돌아가고, 테스트가 통과한다.

## 6. Milestone 3-4 – 인증 & 상태관리 (Auth + Data Fetching)

### 6.1 목표

* **JWT 기반 로그인/로그아웃**을 구현하고,
* **Protected Routes**로 인증이 필요한 페이지를 보호하며,
* **React Query**로 서버 상태를 효율적으로 관리하고,
* **통계 대시보드**를 차트 라이브러리로 구현할 수 있는 것.

이 단계까지가 사실상 "실무에서 바로 써먹을 수 있는 수준"이다.

### 6.2 다루는 기술

* **M3 - 로그인/인증/인가 UX**

  * JWT 저장 (localStorage/sessionStorage)
  * Context API (AuthContext)
  * Protected Routes (useAuth hook)
  * 역할 기반 UI (OWNER/MANAGER/MEMBER)

* **M4 - 통계/데이터 패칭 패턴**

  * React Query (useQuery, useMutation)
  * 캐싱 & Stale Time
  * Optimistic Update
  * Recharts (차트 라이브러리)

### 6.3 이 단계에서 완성해야 할 기능

**M3 최소 기능**:

1. **로그인/로그아웃**

   * `POST /api/auth/login` - JWT 토큰 발급
   * 토큰을 localStorage에 저장
   * 로그아웃 시 토큰 삭제

2. **Protected Routes**

   * 인증되지 않은 사용자는 `/login`으로 리다이렉트
   * 인증된 사용자만 프로젝트/이슈 접근 가능

3. **역할 기반 UI**

   * OWNER만 팀원 삭제 가능
   * MANAGER는 팀원 추가 가능
   * MEMBER는 읽기만 가능

**M4 최소 기능**:

1. **React Query 적용**

   * `useQuery`로 데이터 조회 (캐싱 자동)
   * `useMutation`으로 생성/수정/삭제

2. **통계 대시보드**

   * `GET /api/stats/daily?from=2025-01-01&to=2025-01-31` - 날짜 범위 통계
   * Recharts로 라인 차트/바 차트 렌더링

3. **Optimistic Update**

   * 댓글 작성 시 서버 응답 전에 UI 즉시 업데이트
   * 실패 시 롤백

### 6.4 통과 체크리스트

**M3**:

* [ ] JWT 로그인을 구현하고, 토큰을 저장/사용할 수 있다.
* [ ] Protected Routes로 인증되지 않은 사용자를 리다이렉트할 수 있다.
* [ ] AuthContext로 전역 인증 상태를 관리할 수 있다.
* [ ] 역할 기반 UI로 OWNER/MANAGER/MEMBER별 권한을 구분할 수 있다.

**M4**:

* [ ] React Query로 데이터를 조회하고, 자동 캐싱되는 것을 확인할 수 있다.
* [ ] useMutation으로 생성/수정/삭제를 하고, 캐시를 자동 갱신할 수 있다.
* [ ] Recharts로 통계 데이터를 차트로 시각화할 수 있다.
* [ ] Optimistic Update로 UX를 개선할 수 있다.

## 7. Milestone 5-6 – 고급 패턴 (Search/Performance + Testing)

### 7.1 목표

* **검색 UI**를 디바운스와 함께 구현하고,
* **성능 최적화**를 위해 가상 스크롤과 코드 스플리팅을 적용하며,
* **테스트**를 작성하고 **접근성**을 개선할 수 있는 것.

이 단계는 **선택**이다. 정말로 "프로덕션 수준의 품질"을 원할 때만 들어간다.

### 7.2 다루는 기술

* **M5 - 검색/필터 UI + 성능 기초**

  * 검색 입력 디바운스 (lodash/debounce)
  * 필터링 (카테고리, 가격 범위)
  * 가상 스크롤 (react-window)
  * Code Splitting (React.lazy, Suspense)

* **M6 - 테스트 전략 & E2E & 접근성**

  * Vitest + React Testing Library
  * E2E (Cypress/Playwright)
  * ARIA 레이블
  * 키보드 네비게이션

### 7.3 이 단계에서 완성해야 할 기능

**M5 최소 기능**:

1. **검색 UI**

   * `GET /api/search/products?q=키워드&category=전자제품` - 검색 API
   * 입력 디바운스 (500ms)
   * 검색 결과 목록 표시

2. **성능 최적화**

   * react-window로 긴 목록 가상 스크롤
   * React.lazy + Suspense로 코드 스플리팅

**M6 최소 기능**:

1. **단위 테스트**

   * 컴포넌트 렌더링 테스트
   * 사용자 인터랙션 테스트 (버튼 클릭, 폼 제출)

2. **E2E 테스트**

   * 로그인 → 프로젝트 생성 → 이슈 생성 플로우

3. **접근성**

   * ARIA 레이블 추가
   * 키보드로 네비게이션 가능

### 7.4 통과 체크리스트

**M5**:

* [ ] 검색 입력에 디바운스를 적용하고, API 호출 횟수를 줄일 수 있다.
* [ ] 필터링 UI로 카테고리/가격 범위를 선택하고, 검색 결과를 필터링할 수 있다.
* [ ] react-window로 긴 목록을 가상 스크롤할 수 있다.
* [ ] React.lazy + Suspense로 페이지를 코드 스플리팅할 수 있다.

**M6**:

* [ ] Vitest + React Testing Library로 컴포넌트 단위 테스트를 작성할 수 있다.
* [ ] E2E 테스트로 전체 플로우를 자동 검증할 수 있다.
* [ ] ARIA 레이블을 추가하고, 스크린 리더로 확인할 수 있다.
* [ ] 키보드만으로 애플리케이션을 사용할 수 있다.
